Part 1: Solidify Terminology

  • HTTP: The Hypertext Transfer Protocol (HTTP) is the protocol that is used by the World Wide Web (WWW) to request hypertext documents from one system to another. HTTPS is an encrypted form of HTTP and has become standard.
  • URL: A Universal Resource Locator (URL) is the network address of some resource
  • DNS: The Domain Name System (DNS) is a service that takes a URL and returns the corresponding IP address either from a local lookup table or from a web DNS server.
  • Query String: The query string of a URL allows the browser to pass parameters to the backend server.
  • Example: HTTP Verbs: The two most common HTTP Verbs, or methods, are the GET and the POST methods. The GET method is used to request data from the backend server via the query string while the POST method is used to pass data to the backend server via the request body.
  • HTTP Request: An HTTP Request is a message sent to a server by a client requesting some action be taken. In the case of the GET method, the client is requesting the server return some information. For the POST method, the client is sending some information to the server for the server to process accordingly. The PUT and DELETE methods behave how one would expect: the PUT method updates or adds resources to the system and the DELETE method removes resources from the system.
  • HTTP Response: An HTTP Response is a message sent from a server to a client in response to an HTTP Request. This message includes the status of the response, the response header, and any resources that were requested.
  • HTTP Header: The HTTP Header of an HTTP Message contains the metadata of the HTTP Message. This metadata contains things like authentication information, security information, compression information, and more.
  • Browser URL Process: the following things happen when a user types a URL into a browser:
    1. The browser looks up the IP address from the hostname using the DNS
    2. The browser makes the appropriate HTTP request to that IP address
    3. The server processes the request and sends the appropriate response
    4. The browser creates and loads the DOM based on that response
    5. If the response requires resources from elsewhere, the browser makes the appropriate HTTP requests.
